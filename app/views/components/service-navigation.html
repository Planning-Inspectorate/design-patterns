Work in progress

Use the service navigation component to show the service name and provide links to key sections of the service. It helps users orient themselves and move between pages without cluttering the main header.

  • Use the GOV.UK service navigation for services hosted on GOV.UK domains where the GOV.UK header must remain minimal.
  • Use it for PINS-hosted services to provide consistent navigation across pages.

When not to use this component

  • On single-page or linear journeys where navigation would distract from completing the task.
  • For links that belong in the footer (e.g., Privacy, Accessibility, Cookies).
  • To add GOV.UK branding or crown — this is not part of the service navigation.
  • For global navigation across multiple services. It is only for navigation within one service.

How it works

  • Appears directly below the GOV.UK header or PINS header.
  • Displays the service name (e.g., “Appeal a planning decision”).
  • Provides links to key sections of the service, such as:
    • Dashboard
    • Assigned to me
    • Sign out
  • Keep navigation minimal to avoid distracting users.

Design rules

  • Font: Match the header font (GDS Transport for GOV.UK services, Arial or Sans Serif for PINS services).
  • Colour: Use GOV.UK blue for GOV.UK services; use PINS colour palette for PINS services.
  • Colour palette for PINS services should align with MHCLG colourways.
  • Accessibility: Links must have clear focus states and meet WCAG contrast requirements.
  • Responsive behaviour: Navigation should collapse into a menu on smaller screens.
  • Implementation details will be added once a shared approach is agreed.
