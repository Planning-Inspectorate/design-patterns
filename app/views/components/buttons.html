Use the button component to help users carry out an action like starting an application or saving their information.

Refer to the guidance on GOV.UK buttons for the components themselves.

Primary button

Primary buttons should be used for core actions that will progress the user towards their user journey, and you should avoid having more than one on a page at a time to avoid confusing the user. For example:

  • Save and continue
  • Save and return
  • Sign in
  • Confirm and send
  • Publish

The button text should be sentence case, and should describe the exact action it completes when clicked. For example, if the button does not save any information, do not use 'save'.

The buttons should be left aligned and easily scannable when reviewing the page and not hidden in other components.

Consider where the button should go with regards to the rest of the page content. If they are completing a form, place the button beneath the form content to ensure they do not advance without completing the form. Avoid putting any essential content below a button as it may be missed by a screen reader who is quickly scanning the page and looking to progress with the journey.

If the button is not directly completing the action, consider using a secondary button or a button link. For example, on a project page you may have an 'upload documents' button to take the user to a sub-page to upload documents. The sub-page should use a primary button as it will be the action to add it to the system, whereas the button on the project page is only used to start the process so should be a secondary button.

Secondary buttons should be used for secondary or tertiary actions on the page. Interacting with the secondary buttons will either complete actions on the page or take the user down a sub-journey that when completed will return the user to their previous position on the page.

Secondary actions

They can be used to diverge them onto a new sub-journey, such as to add more information without significantly progressing the key user journey, for example:

  • Add document
  • Add another
  • Upload new version
  • Save as draft
  • Open

Tertiary actions

They can also be used to supplement the core user journey on the page by allowing the user to optionally refine the information on the page to support their needs. For example:

  • Apply filters
  • Find address

Warning button

Warning buttons should be used exclusively for destructive actions. An action can be considered destructive if it is irreversible or may result in the loss of data or significant progress, for example:

  • Remove user
  • Delete document

The warning button text used should be specific and direct, and refer only to the action it will complete for example, "Remove user" or "Delete document".

They should be used sparingly (with no more than one on a page at a time) as the strong red colour can be stress inducing. The warning button should be used only for the decisive final action of a journey, such as on the confirmation page of deleting a document rather than the button on the properties page itself.

Link (button)

{% set exampleHtml1 %} View publishing queue {% endset -%} {% set codeHtml1 %} {{ codeBlock(' View publishing queue ', 'html') }} {% endset -%} {{ govukTabs({ id: "header-tabs-1", items: [ { label: "Example", id: "example1", panel: { html: exampleHtml1 } }, { label: "HTML", id: "html1", panel: { html: codeHtml1 } } ] }) }}

Links may be used on their own or as part of a button group exclusively for supporting navigation without completing an action itself, for example:

  • Skip
  • View publishing queue
  • View decision
  • Download
  • Give your feedback (opens in a new tab)

Situationally, it may be preferable to open the new page in a new tab to ensure the user does not lose information added to the previous page, or if having both pages open is essential to task completion. In this situation the link content should include "(opens in a new tab)" after the main text.

If there are a series of links that all open in a new tab you may opt to make the '(opens in a new tab)' visually hidden with the "govuk-visually-hidden" class tag, and instead mention above the list that each following link opens in a new tab.

If a link is being used on it's own outside of body text, punctuation (such as full stops) is not required and only use sentence case. If it is inside body text you should use punctuation, but the fullstops should be outside of the clickable link, like this.

Start buttons should be used to initiate the start of a new journey. Technically they are link HTML elements as they are only used navigate the user to the beginning of a new journey, and will not complete any actions or submit any form data. For example:

  • Start now

Avoid using more than one per page, and do not use the start button as part of a button group.

Button groups should be used to group multiple actions together that directly relate to one another. If used on a form any buttons should directly relate to the form content and main header.

  • Save and continue | Save as draft
  • Continue | Cancel

They should always be ordered with the primary actions on the left followed by any secondary, then tertiary actions on the right.

Avoid using more than one primary button in a button group.

Avoid using more than one link in a button group. If there are multiple links needed, consider a list component with links.


When not to use this component

Do not use button HTML elements in place of links, and link HTML elements in place of buttons. This can have an impact on screenreader users.

There may be situations where styling a button to look like a link or a link to look like a button is needed, and this is fine when validated with research. However, it is essential that buttons (that do actions) are tagged as HTML buttons elements, and links (that only take people places) are tagged as HTML link elements even when styled otherwise.

Accessiblity

Link colours and button colours must pass contrast checks between the text and the background color.

Text size for links should be standard govuk-body-m or govuk-body-s and should match the size preset of other body text in the same container.
