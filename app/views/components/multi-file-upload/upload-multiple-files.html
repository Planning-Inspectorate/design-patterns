When to use this pattern

Use the multi file upload component when you want users to be able to upload more than one file.

How it works

You can use this component as a Nunjucks macro.

You can access it by copying the multi-file-upload.njk file from the PINS design-patterns repository and pasting into your prototypes' components folder.

Call the Nunjucks item using

{% raw %}{% from "components/multi-file-upload.njk" import uploadArea %}{% endraw %}

It should be nested in a ‘two-thirds’ grid layout.

Error messaging

If a user attempts to continue before all files have finished uploading, display the following error summary and message: "Wait for all files to finish uploading".

If a user attempts to upload an incorrect file format, display the format requirements in the error summary box and the file that failed to upload is shown beneath the upload area with the message 'The selected file must be a [list of file types]​’'.

Design history

Internal Planning Inspectorate users can read how user research influenced our decisions in creating this component.

Accessibility

As part of the Local Plans Alpha, we brought our prototype to the Solirius Accessibility Lab, which uses this multi-file upload component. The lab testing identified 2 key issues:

  • screen readers were not informed of upload status or progress, so screen reader users had no way of knowing when a file had finished uploading
  • the 'Remove' and 'Change' links did not have visually hidden text, so screen reader users could not tell which file each link related to

We made the following changes to the component to fix these issues:

  • added visually hidden text to the 'Remove' link so it announces the associated file name, for example "Remove Receipt-1.pdf"
  • added a visually hidden, aria-live="polite" status region that announces "Upload in progress" and "Upload successful" so screen reader users are told when uploads start and finish

Non-JavaScript uploader

A version of the multi-file uploader needs to be created for situations where javascript is not available. The non-javascript version of the multi-file upload has not been finished yet.

multi-file-upload.njk is JavaScript-dependent by design.

Without JavaScript, it fails functionally because:

  • the visible upload UI is a custom dropzone, not a native control.
  • the native file input is visually hidden and only triggered by JS click/keyboard handlers.
  • file rows, progress (Uploading: %), cancel/remove actions, and error states are all rendered/updated in JS.
  • “continue” logic (blocking while uploading, filtering invalid files) is JS-only.
  • hidden inputs for submitted uploaded filenames are created in JS before submit.

So if JS is off and you render only multi-file-upload.njk, users will not get the intended uploader behaviour. That is why you need the separate non-JS fallback. A file for this is available but has not been finished yet (multi-file-uploader-no-script.njk).
