When to use this component

  • For navigation within a section of a back office service.
  • When users need to move between related pages and return to them frequently.
  • For operational services with multiple tasks, documents or case management areas.
  • Alongside the service navigation component where an additional navigation level is needed.

When not to use this component

  • For primary service navigation.
  • For simple journeys with only a small number of pages.
  • As a substitute for breadcrumbs.
  • To show progress through a task or form.

How it works

  • Appears on the top side of the page underneath the h1.
  • Groups related pages under a section heading.
  • Clearly indicates the current page.
  • Uses the MoJ sub navigation pattern.
  • Uses the Planning Inspectorate teal active indicator (#10847E).

Accessibility

  • Use aria-current="page" on the active link.
  • Ensure active states are not communicated through colour alone.
  • Maintain keyboard access to all navigation items.
  • Meet WCAG 2.2 AA requirements.

Research on this component

Users were able to complete tasks successfully using the sub navigation and found the content structure clear. Some users preferred this approach because it reduced scrolling and made sections easier to locate.

Users preferred horizontal navigation because it felt more familiar and made better use of available screen space.

This component was tested with users of complex back office services where users need to move between sections regularly.
