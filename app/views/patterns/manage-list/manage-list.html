Work in progress

This pattern is used to help users manage and organise lists of items.

When to use this pattern

Use this pattern when users need to:

  • add and remove items from a list
  • organise items in a specific order
  • perform actions on multiple items at once

Basic example

How it works

Users can manage a list using a 'Check your answers'-style component. They can change or remove items by selected the button next to each item.

Users can add items to the list by selecting the 'Add another' link. This will take them to a form where they can enter the details of the item they want to add. Once they have added the item, they will be taken back to the list where they can see the new item added.

Customisation

The 'Manage a list' pattern can be customised to fit the needs of your service. You can change the layout and design of the list, as well as the actions that users can perform on the items in the list.

You can use the sortable table component to allow users to reorder items in the list.

Limiting numbers of items

When the maximum number of items has been reached, you can remove the 'Add another' link.

Accessibility and mobile

  • Make sure that where repeated 'Remove' and 'Change' buttons are used, they are clearly labelled for screen readers and accessible via keyboard navigation.
  • Ensure that the 'Add another' button is easily identifiable and accessible on mobile devices.
  • Ensure that pages are optimised for mobile devices and that all interactive elements are easily accessible on smaller screens.

Research on this component

We are planning to conduct research on this component to better understand how users interact with it in different scenarios. We would particularly like to test:

  • 'Add another' wording, placement and whether it should be a button or link
  • How we prevent users from adding items over a prescribed limit
