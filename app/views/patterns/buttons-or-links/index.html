When should we use buttons? When should we use links?

A button is used to perform an action, such as submitting a form or saving data. A link is used to navigate to another page or resource.

Put simply:

  • a button = action happens immediately
  • a link = takes user somewhere to do the action

Why is this important?

Buttons and links may sometimes look similar, but they communicate different things to users and assistive technologies. This is important for accessibility and for users who rely on assistive technologies.

Assistive technologies announce links and buttons differently.

  • a link communicates, ‘this will take me somewhere else‘
  • a button communicates, ‘this will do something‘

Users rely on these cues to understand what will happen when they interact with a control.

WCAG focuses on the following principles for buttons and links:

  • correct semantics (whether assistive technologies can identify what it is)
  • consistent behaviour

WCAG 2.1 Understanding Name, Role, Value (opens new tab)

If a control is announced as a button but takes the user to another page, or is announced as a link but performs an action immediately, the interface can become confusing. This is why the underlying HTML element matters, not just the styling.
