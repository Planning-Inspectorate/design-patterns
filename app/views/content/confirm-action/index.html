When to use this pattern

Use this pattern to put some friction in a user action, so that the user doesn't take the action by accident, or to make sure they understand the consequences of their action.

How it works

This pattern uses core GDS components, such as headings, paragraphs, buttons and links, to create a confirmation page that asks the user to confirm an action.

If the user confirms they want to proceed, the action is taken. If they don’t, they are returned to the previous page.
